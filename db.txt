Столбцы называют полями (атрибутами), они содержат информацию про свойства объекта

Строки называют записями (кортежи), они сожержат значения конкретного свойства

SELECT [ALL | DISTINCT] (*|столбец1 [, столбец_n])
FROM таблица1 [, таблица_n]
WHERE условие
		AND OR
		выражение [NOT] BETWEEN начальное_значение AND конечное_значение
		выражение [NOT] LIKE шаблон_строки
			% - любое количество символов
			_ - один любой символ
		выражение [NOT] GLOB регулярное_выражение
			* - любое количество символов
			? - соответствует одному символу
			. - соответствует любому одиночному символу
			[символы] - соответствует любому одиночному символу из списка [abc]
			[начальный_символ-конечный_символ] - соответствует любому одиночному символу из заданого диапазона [0-9]
			^ - этот символ используется в начале списка символв и соответствует любому символу, который НЕ входит в список [^0-9]
		IS [NOT] NULL - позволяет выбрать все строки, столбцы которых имеют значение NULL	
		выражение [NOT] IN (набор значений | выражение)	
GROUP BY (группировать по)
HAVING условие
ORDER BY col_name | col_number [ASC | DESC]

Декартовое произведение (все возможные комбинации каждой строки одной таблицы с каждой строкой другой таблицы)

Функции агрегирования:
SUM - сумма  (только с цифровыми полями)
AVG - среднее арифметическое (только с цифровыми полями)
COUNT - количество
MIN - минимальное значение
MAX - максимальное значение


Однострочные подзапросы: =, >, >=, <, <= (возвращает из подзапроса только одну строку)
Многострочные подзапросы: IN (NOT IN) (возвращают более одной строки, список значений)

Операторы BETWEEN, LIKE, IS NULL к подзапросам применять нельзя

В подзапросах нельзя использовать ORDER BY и GROUP BY
